# MediaTagger 开发规格说明

> 本文档为项目内部开发/测试规格，面向贡献者与维护者。

## Background

**MediaTagger** 是一款本地桌面端应用，旨在对本地资源（视频、图片、音频等）进行高效的标签化管理，支持快速查找、分类与批量操作。

## Tech Stack & Architecture

- **框架**: Tauri v2
- **前端**: React + TypeScript + Tailwind CSS
- **后端 (Core)**: Rust + Tokio，通过 Tauri IPC 与前端通信
- **数据库**: 本地 SQLite

## Core Mechanisms

### 1. 唯一标识 (Content-based Identity)

- **主键**: `blake3` 哈希值
- **计算策略**:
  - 小文件（< 10MB）：全量计算哈希
  - 大文件（≥ 10MB）：采样计算（前 1MB + 中间 1MB + 最后 1MB）
- **缓存机制**: 结果存入 SQLite，基于 `modified_time` 判定缓存失效

### 2. 被动修复与生命周期管理

- **标签跟随**: 哈希与内容绑定，文件移动/重命名后标签不丢失
- **软删除**: 路径失效时标记 `status=0`（丢失），文件恢复后标签自动找回

## Test Scenarios

详见原 QA 检查清单：文件移动、重命名、同名不同内容、同内容副本、内容变更、删除恢复、批量搜索等场景。

| 测试要点       | 正确行为           | 常见错误           |
| -------------- | ------------------ | ------------------ |
| 文件移动       | 标签始终跟随文件   | 路径变动导致丢失   |
| 文件重命名     | 标签始终跟随文件   | 文件名变动导致丢失 |
| 同名不同内容   | 标签**不**跟随     | 仅因文件名继承标签 |
| 同内容不同名   | 标签共享/跟随      | 相同文件需重新打标 |
| 文件内容被修改 | 视为新文件         | 旧标签残留         |
| 删除后恢复文件 | 标签自动恢复       | 记录被彻底删除     |
